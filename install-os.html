# How to install Linux and other operating systems (OS) # Glossary OS = operating system. VM = virtual machine. # Linux ## Fedora KDE Version >= 38 Screen recording https://www.youtube.com/watch?v=Doj8VpTLShs * As usual: download the ISO file, using Virtual Machine Manager create a VM named fedora, 10240 GB RAM, 300 GB SSD, use virt, use maximum possible vcpus minus two (the hardware computer has a CPU with 16 threads, so the VM uses 14 vcpus). Start the VM. * In the Fedora installer select custom partitioning, use EXT4 instead of btrfs, do not use a separate /home partition. Create a user named administrator. * Log into KDE Plasma 5 > System Settings > Energy Saving > uncheck "Suspend session Automatically Sleep after 15 min". In KDE Plasma 6 > System Settings > Security & Privacy > Screen Locking > uncheck the checkbox "Lock screeen automatically After". * Survival at the command line. sudo is available, SSH server is already installed but needs to be enabled. `systemctl enable --now sshd`. From now on, I do not need the "Graphical Console" of Virtual Machine Manager. KDE Plasma > Log Out. Connect to the VM using ssh `ssh administrator@192.168.122.106`. Advantages: copy/paste of text works, we can resize the terminal that runs SSH as we want, we can continue to use the web browser of the hardware/host computer. `sudo dnf install git emacs-nox mc`. * Survival in the Graphical User Interface (GUI). Install xrdp server https://nmariusp.github.io/rdp.html . Connect to the VM using RDP `xfreerdp /u:administrator /w:1920 /h:1080 /v:192.168.122.168 /video /rfx /network:lan /gfx /dynamic-resolution /sound:sys:pulse` 
sudo dnf remove kwrite ; sudo dnf install kate git-gui gitk krusader

## Alpine Linux Version >= 3.22 Screen recording https://www.youtube.com/watch?v=d9WXHMfBDT0 I installed Alpine Linux 3.22 using alpine-standard-3.22.1-x86_64.iso. As per https://docs.alpinelinux.org/user-handbook/0.1a/index.html. Boot from DVD. Log in as user root without password. Run "setup-alpine". At "Enter mirror number" write "c" , then when asked again press Enter. This will add the "community" repository and use the default mirror (i.e "1", i.e. the Apline Linux CDN). Create a user named "administrator". This user by default can use "doas". "How would you like to use it?" "sys". I connect to the VM in text user interface mode using ssh: 
ssh administrator@192.68.122.6

doas su

# Surviving at the command line: bash, emacs-nox, mc, git.
apk update
apk upgrade
apk add nano emacs-nox mc git bash
# chsh is not available by default. chsh is part of the shadow package.
apk add shadow
chsh -s /bin/bash
exit
chsh -s /bin/bash
doas su

# Surviving in GUI:
# setup-desktop does not have an option to install KDE Plasma. I install KDE Plasma desktop as per https://wiki.alpinelinux.org/wiki/KDE

# KDE Plasma is Wayland only.
setup-wayland-base # Installs dbus, elogind, eudev, community repository, polkit-elogind.

apk add plasma-desktop-meta # Installs 600 binary packages. Installs SDDM.
apk add khelpcenter konsole firefox kcalc # Install the missing KDE Plasma packages.
rc-update add sddm
rc-service sddm start
reboot

## Windows XP Version x64 with Service Pack 2. As per https://pve.proxmox.com/wiki/Windows_XP_Guest_Notes#Use_VirtIO_Drivers Windows XP has reached end of life (EOL) 2014.04. We use the ISO file "Windows XP Professional x64 Edition with Service Pack 2" from MSDN. We use the virtio drivers for Windows XP ISO file https://fedorapeople.org/groups/virt/virtio-win/deprecated-isos/archives/ . From before 2014.04: virtio-win-0.1-65 from 2013.11. Install Windows XP 64 bit OS VM: 4024 MB RAM, 4 vcpus, 100 GB HDD IDE (PATA). Finish installing the OS. Add hardware: IDE CDROM with virtio-win-0.1-65 ISO file, virtio HDD 20 GB (we will delete this later), virtio NIC. Boot, install missing Windows drivers for these virtio HDD, virtio NIC. Shutdown. Remove virtio HDD, remove IDE HDD make sure not to delete the qcow file, add new virtio HDD using the existing qcow file of the HDD where Windows XP is installed. You can connect using an RDP client. "/dynamic-resolution" does not work. So you need to provide the correct width and height via command line parameters. 
xfreerdp /u:administrator /w:1920 /h:1000 /v:192.168.122.109 /video /rfx /network:lan /gfx
Install software: using the built in Internet Explorer navigate to http://ftp.mozilla.org/pub/firefox/releases/52.9.0esr/win32-sha1/en-US . And download and install the newest version of Mozilla Firefox that still supported Windows XP: version 52.9.0esr. Right click on the KDE Plasma panel > Enter Edit Mode, click with the mouse where it says "Drag to move" and drag to the left margin of the screen. Press the two red Close buttons to exit Edit Mode. In Kate main menu > Settings > Configure Kate > search for "welc" > uncheck the check box "Show welcome view for new windows".
